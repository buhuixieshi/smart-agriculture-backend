### 2.4 灌溉模式设计：自动与手动双模

**设计原则**：系统同时支持自动和手动两种灌溉模式，手动优先级高于自动。

| 对比维度 | 🤖 自动模式 | 🖐 手动模式 |
|----------|------------|------------|
| 触发方式 | 系统根据阈值自动判断 | 用户在页面手动点击 |
| 开启条件 | 湿度连续N次低于下限 + 冷却间隔已过 | 用户任意时间可操作 |
| 关闭条件 | 湿度回升至阈值以上 或 超过最长运行时间 | 用户手动点击停止 或 超时自动停止 |
| 安全保护 | 最长运行时间、冷却间隔、断电默认关闭 | 同自动模式的安全保护 |
| 优先级 | 手动操作会暂停自动判断（冷却期内） | 最高优先级 |
| 日志记录 | 自动标注为 AUTO 触发 | 标注操作人 |

**后端实现要点**：
1. irrigation_strategy 表新增字段：uto_mode（BOOLEAN，是否启用自动模式）、cooldown_minutes（冷却间隔）、consecutive_threshold（连续超限次数）
2. 自动模式判断逻辑：遥测数据入库 → 检查 auto_mode → 判断连续超限次数 → 检查冷却间隔 → 自动下发灌溉指令
3. 手动模式：用户调用 API → 校验权限 → 下发指令 → 记录操作人 → 重置自动模式冷却计时器
4. 安全保护：单次最长运行时间到后，后端自动下发 STOP 命令

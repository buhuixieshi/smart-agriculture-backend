# 害虫识别模型接入反馈处理说明

## 1. 设备端已提供的信息

模型服务地址：

```text
http://192.168.20.84:5000/pest/detect
```

支持害虫列表接口：

```text
http://192.168.20.84:5000/pest/ids
```

模型类型：

```text
llava:7b 视觉大模型 + 颜色启发式回退
```

运行位置：

```text
设备端电脑 192.168.20.84
```

注意：设备端电脑需要保持开机，且 IP 可能因为 DHCP 变化。

## 2. 后端已完成的修改

后端配置已从模拟模型切换为真实模型：

```yaml
pest:
  model:
    enabled: true
    url: http://192.168.20.84:5000/pest/detect
    connect-timeout-seconds: 5
    read-timeout-seconds: 30
```

修改文件：

```text
src/main/resources/application.yml
```

模型返回的 `pestName` 可能是英文，例如：

```json
{
  "pestName": "Aphid"
}
```

后端已处理为：优先使用本地知识库里的中文名返回给前端，例如：

```text
aphid -> 蚜虫
whitefly -> 白粉虱
thrips -> 蓟马
spider_mite -> 红蜘蛛
armyworm -> 粘虫
```

修改文件：

```text
src/main/java/com/agriculture/service/impl/PestServiceImpl.java
```

## 3. 模型接口格式

设备端模型接口：

```http
POST /pest/detect
```

请求格式：

```text
multipart/form-data
```

字段：

```text
file    图片文件，必填
plotId  地块 ID，可选
```

模型返回示例：

```json
{
  "pestId": "aphid",
  "pestName": "Aphid",
  "dangerLevel": "MEDIUM",
  "confidence": 0.85,
  "message": "Vision: Aphid",
  "plotId": "3"
}
```

后端实际需要字段：

```text
pestId
pestName
dangerLevel
confidence
message
```

其中 `plotId` 后端目前不会强依赖，返回里有也不会影响。

## 4. 前端/后端测试方式

### 第一步：确认设备端模型服务能访问

在浏览器或 Postman 访问：

```http
GET http://192.168.20.84:5000/pest/ids
```

期望返回：

```json
{
  "supportedPestIds": [
    "aphid",
    "whitefly",
    "thrips",
    "spider_mite",
    "armyworm"
  ]
}
```

如果这个接口访问不了，说明后端也无法调用模型，需要先让设备端确认：

```text
1. 设备端电脑是否开机
2. IP 是否仍是 192.168.20.84
3. Python 模型服务是否启动
4. 防火墙是否放行 5000 端口
5. 后端电脑和设备端电脑是否在同一网络
```

### 第二步：重启后端

因为 `application.yml` 已修改，必须重启 Spring Boot 后端。

IDEA 启动的话：

```text
停止当前后端 -> 重新 Run SmartAgricultureApplication
```

### 第三步：Postman 调后端害虫识别接口

请求：

```http
POST http://localhost:8080/api/pest/detect
```

Body 选择：

```text
form-data
```

参数：

```text
plotId = 1
file = 选择一张 jpg/png 害虫图片
```

成功接入真实模型时，返回中应出现：

```json
{
  "modelStatus": "MODEL_SERVICE"
}
```

如果看到：

```json
{
  "modelStatus": "MOCK_FALLBACK"
}
```

说明后端没有成功调用到设备端模型服务，自动回退到了模拟识别。

## 5. 为什么可能还是 MOCK_FALLBACK

后端模型调用失败时不会直接报错，而是自动回退模拟识别，避免前端接口不可用。

常见原因：

```text
1. 设备端 IP 变了
2. 设备端 Python 服务没启动
3. Ollama 没启动
4. 模型接口响应超过 30 秒
5. 设备端电脑防火墙拦截 5000 端口
6. 后端电脑无法访问 192.168.20.84
7. 模型返回里没有 pestId
```

设备端反馈的启动方式：

```text
cd D:\软件综合设计\ai-service
python app.py
```

并且需要先启动 Ollama：

```text
ollama.exe serve
```

环境变量：

```text
OLLAMA_MODELS=D:\ollama\models
```

## 7. 前端注意事项

模型推理大约：

```text
20-30 秒/次
```

所以前端上传图片后需要显示 loading 状态。

建议超时提示文案：

```text
害虫识别中，请稍候...
```

如果接口最终返回 `MOCK_FALLBACK`，说明当前是模拟结果，可以提示：

```text
当前使用模拟识别结果
```

如果返回 `MODEL_SERVICE`，说明是真实模型结果。

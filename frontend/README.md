# 智慧农业简易前端

这是根据当前 Java 后端接口写的无依赖静态前端，用于本地完整联调。

## 一、启动前提

后端需要运行在：

```text
http://localhost:8080
```

后端 CORS 当前放行：

```text
http://localhost:5173
http://127.0.0.1:5173
```

所以前端建议用 5173 端口启动。

## 二、启动方式

推荐用 Node 启动：

```powershell
node frontend/server.js
```

也可以用 Python 启动：

```powershell
python -m http.server 5173 -d frontend
```

然后浏览器打开：

```text
http://localhost:5173
```

## 三、已接入功能

- 登录、注册、退出登录；
- 地块列表、新增、修改、删除；
- 设备列表、新增、修改、删除；
- 设备绑定地块、解绑、启用、停用；
- 最新遥测、历史遥测；
- 模拟硬件 HTTP 上报；
- 水泵控制；
- 补光控制；
- 控制记录和命令状态查询；
- 灌溉策略；
- 补光策略；
- 用水限制；
- 告警查询、确认、恢复、关闭；
- 操作日志；
- AI 智能农事问答；
- AI 页面跳转动作；
- AI 硬件控制动作确认执行；
- 害虫图片识别和识别记录。

## 四、推荐联调顺序

1. 打开“登录注册”，先注册或登录。
2. 打开“地块管理”，确认至少有一个地块。
3. 打开“设备管理”，确认至少有一个设备，并绑定到地块。
4. 打开“遥测数据”，点击“模拟 HTTP 上报”，确认后端 `telemetry_data` 有数据。
5. 打开“总览”，查看地块最新遥测和活跃告警。
6. 打开“设备控制”，测试水泵和补光命令。
7. 打开“策略配置”，保存灌溉策略和补光策略。
8. 打开“AI 助手”，测试：

```text
当前地块需要灌溉吗？
跳转到设备管理页面
打开水泵
关闭补光灯
```

## 五、注意事项

- 修改、删除、控制类接口需要登录后的 JWT。
- 如果收到 `401 Unauthorized`，请先登录。
- 如果浏览器提示跨域失败，请确认前端地址是 `http://localhost:5173`。
- AI 返回 `actionProposal` 时，前端会弹确认框后再执行硬件控制。
- 当前前端目标是联调完整功能，不是最终 UI 设计稿。

# 害虫识别模型前端联调说明

更新时间：2026-07-06

## 1. 当前状态

后端已经从模拟害虫识别切换为真实模型服务。

当前模型服务地址：

```text
http://192.168.20.84:5000/pest/detect
```

后端配置：

```yaml
pest:
  model:
    enabled: true
    url: http://192.168.20.84:5000/pest/detect
    connect-timeout-seconds: 5
    read-timeout-seconds: 30
```

后端测试已通过，并且测试日志显示已经成功请求到模型服务：

```text
POST http://192.168.20.84:5000/pest/detect
Response 200 OK
```

## 2. 前端调用接口

前端仍然调用后端接口，不直接调用模型服务。

```http
POST http://localhost:8080/api/pest/detect
Content-Type: multipart/form-data
```

不需要 token。

表单参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| file | File | 是 | 害虫图片，jpg/png |
| plotId | Number/String | 否 | 地块 ID |

示例：

```text
form-data:
plotId = 1
file = leaf.jpg
```

## 3. 返回示例

真实模型识别成功时：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "plotId": 1,
    "fileName": "leaf.jpg",
    "pestId": "aphid",
    "pestName": "蚜虫",
    "dangerLevel": "MEDIUM",
    "confidence": 0.85,
    "modelStatus": "MODEL_SERVICE",
    "detectTime": "2026-07-06T10:30:00",
    "suggestion": {
      "pestId": "aphid",
      "pestName": "蚜虫",
      "dangerLevel": "MEDIUM",
      "description": "...",
      "physicalControl": [],
      "biologicalControl": [],
      "chemicalControl": [],
      "prevention": []
    }
  }
}
```

## 4. 前端重点字段

| 字段 | 说明 |
|---|---|
| pestId | 害虫唯一标识 |
| pestName | 中文害虫名，后端已处理为中文 |
| dangerLevel | 危险等级 |
| confidence | 置信度，0 到 1 |
| modelStatus | 识别来源 |
| suggestion | 防治建议 |

`dangerLevel` 可能值：

```text
LOW
MEDIUM
HIGH
```

`modelStatus` 说明：

```text
MODEL_SERVICE  使用真实模型识别结果
MOCK_FALLBACK  模型不可用时，后端自动使用模拟结果
```

前端可以根据 `modelStatus` 显示提示：

```text
MODEL_SERVICE：真实模型识别
MOCK_FALLBACK：当前使用模拟识别结果
```

## 5. Loading 与超时说明

设备端反馈：模型推理大约需要 20-30 秒。

所以前端上传图片后必须加 loading。

建议文案：

```text
害虫识别中，请稍候...
```

建议前端请求超时时间不要低于 35 秒。

如果前端 HTTP 客户端默认超时较短，需要单独给该接口加长超时时间。

## 6. 英文 pestName 问题已处理

模型端返回的 `pestName` 可能是英文，例如：

```json
{
  "pestName": "Aphid"
}
```

后端已经做了处理：会优先用本地知识库中文名返回给前端。

例如：

```text
aphid       -> 蚜虫
whitefly    -> 白粉虱
thrips      -> 蓟马
spider_mite -> 红蜘蛛
armyworm    -> 粘虫
```

所以前端正常展示 `data.pestName` 即可。

## 7. 支持的害虫类型

当前模型支持：

| pestId | 中文名 | 危险等级 |
|---|---|---|
| aphid | 蚜虫 | MEDIUM |
| whitefly | 白粉虱 | MEDIUM |
| thrips | 蓟马 | MEDIUM |
| spider_mite | 红蜘蛛 | HIGH |
| armyworm | 粘虫 | HIGH |

## 8. 识别历史与统计接口

识别成功后，后端会保存识别历史。

### 识别历史

```http
GET /api/pest/records?plotId=1&pestId=aphid&startDate=2026-07-01&endDate=2026-07-07
```

参数都可选。

### 害虫趋势

```http
GET /api/pest/trend?plotId=1&startDate=2026-07-01&endDate=2026-07-07
```

### 害虫分布

```http
GET /api/pest/distribution?plotId=1&startDate=2026-07-01&endDate=2026-07-07
```

## 9. 前端联调建议顺序

1. 先调用害虫建议列表：

```http
GET /api/pest/suggestions
```

2. 再上传图片：

```http
POST /api/pest/detect
```

3. 看返回：

```text
data.modelStatus 是否为 MODEL_SERVICE
data.pestName 是否为中文
data.suggestion 是否有防治建议
```

4. 再查历史：

```http
GET /api/pest/records?plotId=1
```

5. 最后查趋势和分布：

```http
GET /api/pest/trend?plotId=1
GET /api/pest/distribution?plotId=1
```

## 10. 如果返回 MOCK_FALLBACK

如果前端看到：

```json
{
  "modelStatus": "MOCK_FALLBACK"
}
```

说明后端没有成功拿到真实模型结果，可能原因：

```text
1. 设备端电脑没开机
2. 设备端 IP 变了
3. 模型服务没启动
4. 模型推理超时
5. 后端电脑访问不到 192.168.20.84:5000
6. 模型返回里没有 pestId
```

这种情况下前端接口仍然可用，只是结果为模拟识别。

# 智慧农业项目前后端联调说明（Day5 合并后）

更新时间：2026-07-04

本文档用于前端对接当前后端项目。当前后端是 BE-A 与成员5 BE-B 的 Day5 成果合并版本，已经包含基础业务接口、JWT 鉴权、MQTT 遥测接入、控制命令下发、命令回执处理、灌溉记录、IoTDA 上报入口、低湿度告警规则和设备离线告警规则。

## 1. 基础信息

后端默认地址：

```text
http://localhost:8080
```

如果后端部署在局域网机器上，前端可改为：

```text
http://192.168.20.143:8080
```

当前后端端口来自：

```yaml
server:
  port: 8080
```

当前跨域只放行：

```text
http://localhost:5173
http://127.0.0.1:5173
```

如果前端通过局域网 IP 访问，例如 `http://192.168.20.xxx:5173`，需要后端在 `CorsConfig` 里额外加入该 Origin。

## 2. 通用响应结构

大部分接口返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

常见错误：

```json
{
  "code": 400,
  "message": "deviceCode为必填参数",
  "data": null
}
```

JWT 缺失或失效时，安全过滤器返回 HTTP 401。前端对写操作和控制接口必须携带：

```http
Authorization: Bearer <token>
```

## 9. 控制命令接口

控制接口需要 JWT。

### 9.1 通用控制下发

```http
POST /api/control/send?deviceCode=DEV-A003&commandType=PUMP_ON&commandValue=ON
Authorization: Bearer <token>
```

支持的 `commandType`：

```text
PUMP_ON
PUMP_OFF
LIGHT_ON
LIGHT_OFF
```

`commandValue` 可传：

```text
ON
OFF
```

缺少 `deviceCode` 或 `commandType` 时，后端返回业务码 400，不再返回 500。

成功返回 `ControlCommand`，核心字段：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "commandNo": "CMD20260704103000123",
    "deviceCode": "DEV-A003",
    "commandType": "PUMP_ON",
    "commandValue": "ON",
    "status": "SENT",
    "sentAt": "2026-07-04T10:30:00",
    "ackAt": null,
    "errorMessage": null
  }
}
```

状态语义：

```text
PENDING = 已写入数据库，尚未发送
SENT    = 已发送到 MQTT Broker，不代表硬件执行成功
SUCCESS = 收到硬件成功回执
FAILED  = 收到硬件失败回执，或发送失败
```

### 9.2 灌溉控制 JSON 接口

```http
POST /api/control/irrigation
Authorization: Bearer <token>
Content-Type: application/json
```

```json
{
  "deviceCode": "DEV-A003",
  "action": "ON"
}
```

`action` 只支持：

```text
ON
OFF
```

内部映射：

```text
ON  -> PUMP_ON
OFF -> PUMP_OFF
```

### 9.3 查询单条命令状态

```http
GET /api/control/commands/{commandNo}
Authorization: Bearer <token>
```

前端建议在下发命令后轮询该接口，直到状态变成 `SUCCESS` 或 `FAILED`。

### 9.4 查询设备控制记录

```http
GET /api/control/list?deviceCode=DEV-A003
Authorization: Bearer <token>
```

`deviceCode` 是必填参数。缺参时返回：

```json
{
  "code": 400,
  "message": "deviceCode为必填参数",
  "data": null
}
```

## 10. 灌溉记录接口

### 10.1 按设备查询灌溉记录

```http
GET /api/irrigation/list?deviceCode=DEV-A003
```

`deviceCode` 是必填参数。缺参时返回业务码 400。

灌溉记录生成规则：

```text
PUMP_ON  下发成功后创建 RUNNING 记录
PUMP_OFF 下发成功后结束最近一条 RUNNING 记录，改为 FINISHED
```

当前 `waterAmount` 暂不计算，因为没有流量计实测数据。

## 11. 硬件 / MQTT 对接

当前 MQTT 配置：

```yaml
mqtt:
  broker-url: tcp://127.0.0.1:1883
  client-id: smart-agriculture-backend
  username: fasong
  password: fasong123
  qos: 1
  topics:
    telemetry: device/data/upload
    heartbeat: device/heartbeat
    control: device/control/cmd
    control-reply: device/control/reply
```

### 11.1 遥测上报 Topic

```text
device/data/upload
```

示例：

```json
{
  "deviceId": "DEV-A001",
  "timestamp": 1780000000000,
  "data": {
    "soilMoisture": 35.5,
    "temperature": 27.3,
    "humidity": 65.1,
    "illuminance": 850,
    "pumpStatus": "OFF",
    "light_status": "ON"
  }
}
```

后端兼容字段：

```text
deviceId / deviceCode
soilMoisture / soil_moisture
temperature / airTemperature / air_temperature / Temp
humidity / airHumidity / air_humidity / Humi
illuminance / lux / lightIntensity / LightIntensity
pumpStatus / pump_status / motor_status / motorStatus / MotorStatus
lightStatus / light_status / LightStatus / light_status_value
```

处理结果：

```text
1. 写入 telemetry_data
2. 更新 device.status = ONLINE
3. 更新 device.last_heartbeat
4. 推送 WebSocket TELEMETRY
5. 检查低湿度告警规则
```

### 11.2 心跳 Topic

```text
device/heartbeat
```

示例：

```json
{
  "deviceId": "DEV-A001"
}
```

收到心跳后，后端会更新设备在线状态和最后心跳时间。

设备 90 秒没有心跳，会被定时任务标记为 `OFFLINE`，并生成 `DEVICE_OFFLINE` 告警记录。

### 11.3 控制命令下发 Topic

```text
device/control/cmd
```

当前后端在调用 `/api/control/send` 或 `/api/control/irrigation` 后，会发布 MQTT 命令到该 Topic。

### 11.4 命令回执 Topic

```text
device/control/reply
```

硬件或 MQTTX 可发布：

```json
{
  "commandId": "CMD20260704103000123",
  "status": "SUCCESS",
  "message": "motor on ok"
}
```

失败示例：

```json
{
  "commandId": "CMD20260704103000123",
  "status": "FAILED",
  "message": "motor timeout"
}
```

后端会更新：

```text
control_command.status
control_command.ack_at
control_command.error_message
```

前端确认硬件最终执行结果的推荐方式：

```text
1. 查询 GET /api/control/commands/{commandNo}
2. 或查询 GET /api/control/list?deviceCode=DEV-A003
3. 同时观察后续真实遥测中的 pumpStatus / lightStatus
```

```json

  "deviceId": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",

    "temperature": 27.6,
    "humidity": 62.3,
    "illuminance": 850,
    "light_status": "ON",
    "motor_status": "OFF"
  }

```

成功返回：

```json

  "code": 200,
```

## 14. 当前可优先联调的闭环

### 14.1 水泵控制闭环

推荐设备：

```text
DEV-A003 / PUMP_CONTROLLER
```

流程：

```text
1. 前端登录拿 token
2. POST /api/control/send?deviceCode=DEV-A003&commandType=PUMP_ON&commandValue=ON
3. 后端返回 commandNo 和 SENT
4. MQTT Broker 收到 device/control/cmd
5. 硬件执行并向 device/control/reply 回执 SUCCESS/FAILED
6. 前端轮询 GET /api/control/commands/{commandNo}
7. 查询 GET /api/control/list?deviceCode=DEV-A003
8. 查询 GET /api/irrigation/list?deviceCode=DEV-A003
9. 安全测试 PUMP_OFF
```

### 14.2 真实遥测入库闭环

流程：

```text
1. 硬件向 device/data/upload 发布遥测，或调用 POST /api/iotda/report
2. 后端写入 telemetry_data
3. GET /api/telemetry/latest?plotId=1 可看到最新数据
4. GET /api/telemetry/history?plotId=1&page=1&size=20 可看到历史数据
5. WebSocket /ws/realtime 收到 TELEMETRY
6. GET /api/devices 可看到设备状态和 lastHeartbeat 更新
```


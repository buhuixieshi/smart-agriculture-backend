# 模拟传感器与异常告警联调说明（给前端）

## 一、本轮后端改动概览

本轮主要围绕模拟传感器联调、地块 2 数据接入、设备在线/离线展示、传感器异常告警做了后端适配。

当前支持：

1. MQTT 可接收更多遥测 Topic。
2. 模拟设备可自动绑定到对应地块。
3. 设备收到数据后自动标记 `ONLINE`。
4. 设备超过约 90 秒未上报后自动标记 `OFFLINE`。
5. 土壤湿度、空气温度、空气湿度、光照强度异常值会触发告警。
6. 异常数据不会写入 `telemetry_data`，避免数据库字段范围错误。
7. 告警会通过 WebSocket 推送 `type=ALARM`。
8. 设备上下线会通过 WebSocket 推送 `type=DEVICE_STATUS`。

## 二、MQTT Topic 变化

后端现在订阅：

```text
device/data/#
```

因此以下 Topic 都可以被后端接收：

```text
device/data/upload
device/data/upload/humidity
device/data/upload2
```

当前模拟地块 2 可以继续使用：

```text
device/data/upload2
```

## 三、地块 2 模拟传感器上报格式

推荐 JSON：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "soilMoisture": 35.54,
  "airTemperature": 28.23,
  "airHumidity": 59.68,
  "illuminance": 426.26,
  "pumpStatus": "OFF",
  "lightStatus": "OFF"
}
```

说明：

- `deviceCode` 后缀 `_002` 会被后端识别为地块 2。
- 如果 `device` 表中没有该设备，后端会自动创建模拟设备。
- 自动创建的设备信息大致为：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "deviceName": "模拟传感器-2",
  "deviceType": "BEARPI_SIMULATOR",
  "plotId": 2,
  "status": "ONLINE"
}
```

## 四、设备在线/离线状态

### 1. 上报后在线

模拟器只要持续上报遥测数据，后端会更新：

```text
device.status = ONLINE
device.last_heartbeat = 当前时间
```

同时通过 WebSocket 推送：

```json
{
  "type": "DEVICE_STATUS",
  "plotId": 2,
  "data": {
    "deviceId": 8,
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
    "plotId": 2,
    "status": "ONLINE",
    "lastHeartbeat": "2026-07-09T10:40:00"
  }
}
```

### 2. 超时后离线

如果设备约 90 秒没有新的遥测或心跳，后端定时任务会标记：

```text
device.status = OFFLINE
```

并推送：

```json
{
  "type": "DEVICE_STATUS",
  "plotId": 2,
  "data": {
    "deviceId": 8,
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
    "plotId": 2,
    "status": "OFFLINE",
    "lastHeartbeat": "2026-07-09T10:40:00"
  }
}
```

前端设备管理页、大屏设备状态卡片可以监听该消息实时刷新。

也可以通过接口查询：

```http
GET /api/devices?plotId=2
```

## 五、传感器异常告警范围

后端现在对以下指标做异常范围判断：

| 字段 | 正常范围 | 异常告警类型 |
| --- | --- | --- |
| `soilMoisture` | `0 ~ 100` | `SOIL_MOISTURE_ABNORMAL` |
| `airTemperature` | `-10 ~ 80` | `AIR_TEMPERATURE_ABNORMAL` |
| `airHumidity` | `0 ~ 100` | `AIR_HUMIDITY_ABNORMAL` |
| `illuminance` | `0 ~ 1000` | `ILLUMINANCE_ABNORMAL` |

异常值会立即生成 `CRITICAL` 告警。

异常数据不会写入 `telemetry_data`，避免出现数据库字段溢出。

## 七、WebSocket 告警消息

异常告警生成后，后端会推送：

```json
{
  "type": "ALARM",
  "plotId": 2,
  "data": {
    "id": 16,
    "plotId": 2,
    "deviceId": 8,
    "alarmType": "AIR_TEMPERATURE_ABNORMAL",
    "severity": "CRITICAL",
    "triggerValue": 10000,
    "thresholdValue": 80,
    "status": "ACTIVE",
    "message": "空气温度数据异常，正常范围应为-10到80。",
    "createTime": "2026-07-09T11:00:00"
  }
}
```

前端建议：

- 收到 `type=ALARM` 后刷新告警列表。
- 根据 `severity=CRITICAL` 显示红色高危提醒。
- 根据 `alarmType` 显示具体指标名称。

## 八、告警列表查询

查询地块 2 所有告警：

```http
GET /api/alarms?plotId=2
```

按类型查询：

```http
GET /api/alarms?plotId=2&alarmType=SOIL_MOISTURE_ABNORMAL
```

按状态查询：

```http
GET /api/alarms?plotId=2&status=ACTIVE
```

按等级查询：

```http
GET /api/alarms?plotId=2&severity=CRITICAL
```

## 九、仍保留的已有告警

除了本轮新增的传感器异常告警，后端仍保留：

| 告警类型 | 说明 |
| --- | --- |
| `LOW_SOIL_MOISTURE` | 土壤湿度连续低于阈值 |
| `DEVICE_OFFLINE` | 设备超时未上报 |
| `WATER_DAILY_LIMIT` | 今日用水量达到上限 |
| `WATER_MONTHLY_LIMIT` | 本月用水量达到上限 |
| `WATER_SINGLE_LIMIT` | 单次灌溉用水量超过上限 |
| `WATER_DAILY_WARNING` | 今日用水量接近上限 |
| `WATER_MONTHLY_WARNING` | 本月用水量接近上限 |

## 十、注意事项

1. 后端改动后需要重启 Spring Boot，MQTT 新订阅才会生效。
2. 异常传感器数据不会写入 `telemetry_data`，但会生成 `alarm`。
3. 合法传感器数据会正常写入 `telemetry_data` 并推送 `type=TELEMETRY`。
4. 设备上下线建议同时监听 WebSocket `DEVICE_STATUS`，并保留接口轮询兜底。

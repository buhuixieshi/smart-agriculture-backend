# 设备与地块自由绑定接口说明

## 一、改动概述

后端已支持前端自由选择设备绑定到指定地块，也支持解绑设备与地块的关系。

该能力主要用于设备管理页面：

- 前端可展示所有设备。
- 前端可展示所有地块。
- 用户可选择某个设备绑定到某个地块。
- 用户可将设备从当前地块解绑。

## 二、推荐接口

### 1. 更新设备地块绑定

```http
PUT /api/devices/binding
Content-Type: application/json
Authorization: Bearer token
```

### 2. 按设备 ID 绑定

```json
{
  "deviceId": 12,
  "plotId": 2
}
```

含义：

- 将数据库设备 ID 为 `12` 的设备绑定到地块 `2`。

### 3. 按设备编号绑定

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "plotId": 2
}
```

含义：

- 将设备编号为 `6a44b8fdcbb0cf6bb96ad1a1_bearpi_002` 的设备绑定到地块 `2`。

## 三、解绑设备

### 1. 按设备 ID 解绑

```json
{
  "deviceId": 12,
  "plotId": null
}
```

### 2. 按设备编号解绑

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "plotId": null
}
```

含义：

- `plotId` 传 `null` 时，后端会将该设备从当前地块解绑。

## 四、返回示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 12,
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
    "deviceName": "模拟传感器-2",
    "deviceType": "BEARPI_SIMULATOR",
    "plotId": 2,
    "status": "ONLINE",
    "lastHeartbeat": "2026-07-09T15:03:23",
    "signalStrength": null,
    "battery": null,
    "createdAt": "2026-07-09T10:30:00",
    "updatedAt": "2026-07-09T15:10:00"
  }
}
```

## 五、兼容保留接口

原有接口仍然保留，可以继续使用：

```http
PUT /api/devices/{id}/bind?plotId=2
PUT /api/devices/{id}/unbind
```

但是建议前端优先使用新的：

```http
PUT /api/devices/binding
```

原因：

- 支持 `deviceId`。
- 支持 `deviceCode`。
- 支持绑定和解绑共用一个接口。
- 更适合前端设备管理页面的自由选择交互。

## 六、前端推荐交互

设备管理页面可以按以下方式设计：

1. 请求设备列表：

```http
GET /api/devices
```

2. 请求地块列表：

```http
GET /api/plots
```

3. 用户在设备行中选择目标地块。

4. 调用绑定接口：

```http
PUT /api/devices/binding
```

请求体：

```json
{
  "deviceId": 12,
  "plotId": 2
}
```

5. 绑定成功后刷新设备列表。

## 七、注意事项

### 1. MQTT 上报建议带 deviceCode

模拟器或硬件上报数据时，建议始终带上 `deviceCode`：

```json
{
  "deviceId": "2",
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "soilMoisture": 36.54,
  "airTemperature": 28.53,
  "airHumidity": 61.45,
  "illuminance": 403.57,
  "pumpStatus": "OFF",
  "lightStatus": "OFF"
}
```

后端会优先按照 `deviceCode` 查找设备。

### 2. 不建议只传 deviceId

如果 MQTT 只传：

```json
{
  "deviceId": "2"
}
```

后端可能会把 `"2"` 当作数据库设备主键去查，容易和真实设备关系错位。

因此设备上报建议保留：

```json
{
  "deviceCode": "真实设备编号"
}
```

### 3. 绑定关系以数据库为准

设备数据最终属于哪个地块，取决于 `device` 表中该设备当前绑定的 `plot_id`。

前端修改绑定后，后续该设备的遥测数据会按新的地块关系入库和展示。

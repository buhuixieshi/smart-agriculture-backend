## 8. 控制接口

控制接口都需要 token。

### 通用控制下发

```http
POST /api/control/send?deviceCode=6a44b8fdcbb0cf6bb96ad1a1_bearpi_001&commandType=PUMP_ON&commandValue=ON

```

支持命令：

```text

PUMP_OFF   关水泵
LIGHT_ON   开补光灯
```

状态说明：

```text
PENDING  已写库，准备发送
SENT     已发送到 MQTT Broker
SUCCESS  收到设备成功回复或硬件网关同步成功
FAILED   下发失败

```

## 9. 补光功能

### 查询补光状态

```http
GET /api/light/status?plotId=1
```

或：

```http
GET /api/light/status?deviceCode=6a44b8fdcbb0cf6bb96ad1a1_bearpi_001
```

返回重点字段：

```text
illuminance          当前光照
lightStatus          当前灯状态
autoMode             自动补光是否开启
illuminanceMin       低于该值自动开灯
illuminanceMax       高于该值自动关灯
latestCommandStatus  最新灯命令状态
```

### 手动控制补光灯

```http
POST /api/light/control
Authorization: Bearer <token>
Content-Type: application/json
```

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "ON",
  "force": false
}
```

说明：

```text
action 支持 ON / OFF
开灯时如果当前光照已经高于 illuminanceMax，后端会拒绝
如必须强制开灯，传 force=true
```

### 查询补光策略列表

```http
GET /api/light-strategies
```

### 查询某地块补光策略

```http
GET /api/light-strategies/{plotId}
```

### 修改某地块补光策略

```http
PUT /api/light-strategies/{plotId}
Authorization: Bearer <token>
Content-Type: application/json
```

```json
{
  "illuminanceMin": 500,
  "illuminanceMax": 800,
  "autoMode": true,
  "startTime": "06:00:00",
  "endTime": "20:00:00",
  "cooldownMinutes": 5
}
```

自动补光逻辑：

```text
低于 illuminanceMin 且灯未开，自动下发 LIGHT_ON
高于 illuminanceMax 且灯未关，自动下发 LIGHT_OFF
不在 startTime/endTime 时间段内，如果灯为 ON，会自动下发 LIGHT_OFF
cooldownMinutes 内不会重复下发灯命令
```

## 17. MQTT 与设备控制

当前 MQTT 配置：

```yaml
mqtt:
  enabled: true
  broker-url: tcp://127.0.0.1:1883
  client-id: smart-agriculture-backend
  username: fasong
  password: fasong123
  qos: 1
  topics:
    telemetry: device/data/upload
    heartbeat: device/heartbeat
    control: device/control/cmd
    control-reply: device/control/reply
```

后端下发控制命令到：

```text
device/control/cmd
```

设备端执行后回复到：

```text
device/control/reply
```

回复格式：

```json
{
  "commandId": "CMD20260705120000001",
  "status": "SUCCESS",
  "message": "pump on ok"
}
```

`commandId` 对应后端返回的 `commandNo`。

# 后端连接设备端测试说明

日期：2026-07-04

本文档用于测试当前后端是否已经通过 `server.js` 成功控制 BearPi。

## 1. 测试前确认

需要启动：

```text
1. MySQL
2. 后端 Spring Boot，端口 8080
3. 成员6电脑上的 server.js，端口 3000
4. BearPi 已连接 COM10
```

后端配置确认：

```yaml
hardware:
  control:
    enabled: true
    base-url: http://192.168.20.190:3000
    device: bearpi_001
```

先直接测试 server.js：

```http
GET http://192.168.20.190:3000/api/device/status
```

能返回温湿度、光照、水泵状态，说明 server.js 和 BearPi 通。

## 2. 登录获取 token

```http
POST http://localhost:8080/api/auth/login
Content-Type: application/json
```

请求体示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

从返回值中复制：

```text
data.token
```

后续控制接口都要带：

```http
Authorization: Bearer <token>
```

## 3. 测试打开水泵

```http
POST http://localhost:8080/api/control/send?deviceCode=6a44b8fdcbb0cf6bb96ad1a1_bearpi_001&commandType=PUMP_ON&commandValue=ON
Authorization: Bearer <token>
```

成功响应中应看到：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
    "commandType": "PUMP_ON",
    "commandValue": "ON",
    "status": "SUCCESS",
    "errorMessage": "ok",
    "ackAt": "非空"
  }
}
```

成功标准：

```text
1. status = SUCCESS
2. ackAt 不为空
3. errorMessage = ok
4. BearPi 水泵实际打开
```

## 4. 查询命令状态

从上一步响应中复制 `commandNo`，查询：

```http
GET http://localhost:8080/api/control/commands/{commandNo}
Authorization: Bearer <token>
```

示例：

```http
GET http://localhost:8080/api/control/commands/CMD20260704140554471
```

期望：

```text
status = SUCCESS
ackAt 不为空
```

## 5. 查看设备最新状态

调用设备端：

```http
GET http://192.168.20.190:3000/api/device/status
```

期望：

```json
{
  "motorStatus": "ON"
}
```

也可以等待下一条遥测上报后查后端：

```http
GET http://localhost:8080/api/telemetry/latest?plotId=1
```

期望：

```text
pumpStatus = ON
```

## 8. 常见问题

### 8.1 返回 FAILED，提示连接失败

检查：

```text
1. server.js 是否启动。
2. IP 是否还是 192.168.20.190。
3. 后端机器能否访问 http://192.168.20.190:3000/api/device/status。
4. application.yml 中 hardware.control.base-url 是否正确。
```

### 8.2 后端 status=SUCCESS，但硬件没动作

说明 server.js 返回了成功，但串口或 BearPi 执行可能有问题。

检查：

```text
1. server.js 控制日志。
2. COM10 是否连接正确。
3. server.js 是否真的把 motor/light 指令发给 BearPi。
4. 直接调用 server.js /api/control 是否能控制硬件。
```

### 8.3 遥测没有更新

检查：

```text
1. server.js 是否持续 POST 到 /api/iotda/report。
2. 请求体是否包含 device_code=bearpi_001。
3. 后端 /api/telemetry/latest?plotId=1 是否更新。
4. WebSocket /ws/realtime 是否收到 TELEMETRY。
```

## 9. 当前链路说明

当前项目采用：

```text
遥测上报：BearPi -> COM10 -> server.js -> 后端 /api/iotda/report
控制下发：前端/Postman -> 后端 /api/control/send -> server.js /api/control -> COM10 -> BearPi
```

MQTT 控制仍保留为备用模式。若要切回 MQTT，将：

```yaml
hardware.control.enabled: false
```

即可让后端重新使用旧的 MQTT `device/control/cmd` 下发方式。

# 控制接口增强说明

## 一、本次后端修改结论

后端已按前端控制能力反馈和设备端补充说明，扩展控制命令能力：

- 灌溉控制支持 `durationSeconds`
- 补光控制支持 `brightness`
- 补光控制支持 `durationSeconds`
- 通用控制接口 `/api/control/send` 也支持 `durationSeconds` 和 `brightness`
- 命令状态查询会返回 `durationSeconds` 和 `brightness`
- 硬件 HTTP 控制地址已改为 `http://192.168.20.84:3000`
- 补光灯最终发给硬件的 `action` 使用前端传入的亮度值

## 二、数据库变更

已有数据库需要执行：

```sql
src/main/resources/sql/alter_control_command_duration_brightness.sql
```

新增字段：

```sql
duration_seconds INT
brightness INT
```

完整建表脚本 `schema.sql` 也已同步更新。

## 三、灌溉控制

### 接口

```http
POST /api/control/irrigation
Authorization: Bearer JWT
Content-Type: application/json
```

### 请求示例

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "ON",
  "durationSeconds": 600
}
```

关闭：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "OFF"
}
```

### 说明

- `durationSeconds` 只在 `action=ON` 时记录并下发。
- 后端会把该字段保存到 `control_command.duration_seconds`。
- 如果走硬件 HTTP 网关，会把 `durationSeconds` 放进请求体。

## 四、补光控制

### 接口

```http
POST /api/light/control
Authorization: Bearer JWT
Content-Type: application/json
```

### 推荐请求

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "ON",
  "brightness": 80,
  "durationSeconds": 900
}
```

### 兼容请求

也兼容前端直接把亮度放在 `value`：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "ON",
  "value": "80",
  "durationSeconds": 900
}
```

也兼容直接把亮度放在 `action`：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "80",
  "durationSeconds": 900
}
```

关闭补光：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "OFF"
}
```

或者：

```json
{
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
  "action": "0"
}
```

## 五、后端发给硬件的格式

设备端接口：

```http
POST http://192.168.20.84:3000/api/control
Content-Type: application/json
```

补光示例：

```json
{
  "device": "bearpi_001",
  "type": "light",
  "action": "80",
  "durationSeconds": 900,
  "brightness": 80
}
```

其中：

- `action` 会使用前端传入的亮度值。
- 如果前端传 `brightness=80`，后端会把硬件 `action` 设置为 `"80"`。
- 如果前端只传 `action=ON`，后端默认转换为 `"100"`。
- 如果前端只传 `action=OFF`，后端默认转换为 `"0"`。

## 六、命令状态查询

### 接口

```http
GET /api/control/commands/{commandNo}
Authorization: Bearer JWT
```

### 返回示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "commandNo": "CMD20260710120000001",
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_001",
    "commandType": "LIGHT_ON",
    "commandValue": "80",
    "durationSeconds": 900,
    "brightness": 80,
    "status": "SUCCESS",
    "requestSource": "WEB",
    "errorMessage": "ok"
  }
}
```

## 七、通用控制接口

如果前端仍使用通用接口，也已支持新字段：

```http
POST /api/control/send?deviceCode=xxx&commandType=LIGHT_ON&commandValue=80&brightness=80&durationSeconds=900
Authorization: Bearer JWT
```

灌溉：

```http
POST /api/control/send?deviceCode=xxx&commandType=PUMP_ON&commandValue=ON&durationSeconds=600
Authorization: Bearer JWT
```

## 八、当前限制

后端已经记录并下发 `durationSeconds`。

但设备是否真的按时自动关闭，取决于成员6硬件服务是否实现该字段。当前设备端最新补充明确支持补光亮度 `action: "0" ~ "100"`；`durationSeconds` 已由后端透传给硬件服务，需设备端确认是否实际执行自动关闭。

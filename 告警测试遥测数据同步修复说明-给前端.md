# 告警测试遥测数据同步修复说明

## 一、处理背景

前端反馈：使用模拟传感器上传异常土壤湿度时，告警列表可以看到 `SOIL_MOISTURE_ABNORMAL`，但实时监控页仍显示正常湿度。

原因是后端之前对异常遥测数据的处理逻辑是：

- 检测到异常值；
- 创建或刷新告警；
- 不保存到 `telemetry_data`；
- 不推送实时遥测 WebSocket。

所以前端会出现：

- 告警页能看到异常触发值；
- `/api/plots/{id}/latest` 仍返回上一条正常遥测；
- 实时监控页看不到本次异常测试值。

## 二、本次后端修复内容

后端已调整异常遥测处理逻辑。

现在模拟器或设备上传异常遥测时，如果数值在数据库字段可保存范围内，后端会同时执行：

1. 写入 `telemetry_data`
2. 推送实时遥测 WebSocket
3. 创建或刷新 `alarm` 告警记录
4. 让 `/api/plots/{id}/latest` 返回最新异常遥测

例如上传：

```json
{
  "deviceId": "2",
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "soilMoisture": -40,
  "airTemperature": 28.5,
  "airHumidity": 60,
  "illuminance": 430,
  "pumpStatus": "OFF",
  "lightStatus": "OFF"
}
```

后端现在会：

- 保存 `soil_moisture = -40`
- 触发 `SOIL_MOISTURE_ABNORMAL`
- 实时监控页可以拿到最新异常湿度
- 告警页可以看到对应告警

## 三、当前异常阈值规则

当前后端统一使用以下遥测合法范围：

| 指标 | 正常范围 | 告警类型 |
|---|---:|---|
| `soilMoisture` | `0 ~ 100` | `SOIL_MOISTURE_ABNORMAL` |
| `airTemperature` | `-10 ~ 80` | `AIR_TEMPERATURE_ABNORMAL` |
| `airHumidity` | `0 ~ 100` | `AIR_HUMIDITY_ABNORMAL` |
| `illuminance` | `0 ~ 1000` | `ILLUMINANCE_ABNORMAL` |

低于或高于范围都会触发异常告警。

## 四、前端复测方式

### 1. 模拟器通过 MQTT 上传异常土壤湿度

Topic：

```text
device/data/upload2
```

Payload：

```json
{
  "deviceId": "2",
  "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
  "soilMoisture": -40,
  "airTemperature": 28.5,
  "airHumidity": 60,
  "illuminance": 430,
  "pumpStatus": "OFF",
  "lightStatus": "OFF"
}
```

### 2. 检查实时监控接口

请求：

```http
GET /api/plots/2/latest
```

预期：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "plotId": 2,
    "deviceCode": "6a44b8fdcbb0cf6bb96ad1a1_bearpi_002",
    "soilMoisture": -40
  }
}
```

实际返回字段会包含更多遥测字段，前端重点检查 `soilMoisture` 是否为最新异常值。

### 3. 检查告警列表

预期能看到：

```text
SOIL_MOISTURE_ABNORMAL
```

触发值应接近本次上传的异常湿度，例如 `-40`。

### 4. 检查 WebSocket 实时推送

后端会继续推送实时遥测消息，前端如果监听了遥测 WebSocket，应能收到本次异常遥测。

## 五、重要说明

为了避免数据库字段溢出，后端只保存数据库字段范围内的异常值。

例如：

- `soilMoisture = -40`：会保存，会推送，会告警。
- `soilMoisture = 100`：正常边界值，不告警。
- `soilMoisture = 10000`：会告警，但不会写入 `telemetry_data`，因为当前数据库字段 `DECIMAL(5,2)` 存不下 `10000`。

这种处理是为了避免异常测试值导致数据库插入失败。

## 六、对自动控制的影响

异常遥测数据现在虽然会保存给前端展示，但不会触发自动灌溉或智能补光。

也就是说：

- `soilMoisture = -40` 会触发告警；
- 会进入实时监控；
- 但不会被自动灌溉策略当作真实低湿度去误开水泵。

这可以避免告警测试数据影响真实硬件控制。

## 七、后端当前结论

本次前端反馈的问题已处理：

- 告警数据和实时遥测数据已同步；
- `/api/plots/{id}/latest` 可以返回最新异常遥测；
- WebSocket 可以推送异常遥测；
- 告警仍正常生成；
- 自动控制已跳过异常遥测，避免误操作硬件。

# 人脸登录接口更新说明

## 一、本次调整结论

后端已移除旧的“输入用户名 + 上传人脸图片”的人脸登录接口。

前端后续只需要使用新的无用户名人脸登录接口：

```http
POST /api/auth/face/login-auto
```

该接口只上传人脸图片，不再传 `username`。

## 二、已废弃接口

以下接口后端已删除：

```http
POST /api/auth/face/login
```

旧请求方式：

```text
username = 用户名
file = 人脸图片
```

该方式不再使用，前端请不要继续调用。

## 三、当前人脸登录接口

### 接口地址

```http
POST /api/auth/face/login-auto
```

### 鉴权要求

不需要登录，不需要携带 token。

### 请求格式

`multipart/form-data`

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | File | 是 | 用户当前拍摄或上传的人脸图片 |

### 请求示例

```text
file = face.jpg
```

注意：不需要传 `username`。

## 四、后端识别逻辑

后端收到图片后，会执行以下流程：

1. 查询 `user_face` 表中所有已绑定人脸的用户。
2. 调用 Python 人脸识别服务：

```http
POST /face/distance
```

3. 将当前上传的人脸图片与每个已保存的人脸特征逐个计算距离。
4. 找到距离最小的用户。
5. 如果最小距离小于或等于阈值 `0.55`，登录成功。
6. 如果没有匹配用户，返回识别失败。

当前阈值：

```text
distance <= 0.55
```

## 五、成功响应

登录成功时返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "jwt token",
    "userId": 1,
    "username": "test",
    "nickname": "测试用户",
    "role": "USER"
  }
}
```

前端拿到 `data.token` 后，后续接口继续按原来的 Bearer Token 方式调用。

## 六、失败响应

### 1. 未上传图片

```json
{
  "code": 400,
  "message": "face image is required",
  "data": null
}
```

### 2. 系统中没有已绑定人脸用户

```json
{
  "code": 400,
  "message": "no registered face users",
  "data": null
}
```

### 3. 没有识别到已绑定用户

```json
{
  "code": 400,
  "message": "no registered user recognized",
  "data": null
}
```

### 4. 人脸模型服务不可用

```json
{
  "code": 500,
  "message": "face model service request failed: ...",
  "data": null
}
```

## 七、前端需要调整的点

1. 删除或停用旧的人脸登录表单里的用户名输入框。
2. 人脸登录时只上传图片字段 `file`。
3. 接口地址改为：

```http
POST /api/auth/face/login-auto
```

4. 不要再调用：

```http
POST /api/auth/face/login
```

5. 登录成功后的 token 处理逻辑不变。

## 八、仍然保留的相关接口

### 账号密码登录

```http
POST /api/auth/login
```

### 人脸绑定

```http
POST /api/auth/face/register
```

该接口仍然需要用户已登录，因为绑定人脸必须知道当前登录用户是谁。

### 查询用户是否已绑定人脸

```http
GET /api/auth/face/status?username=xxx
```

该接口暂时保留，用于前端判断某个账号是否已绑定人脸。

## 九、后端当前状态

后端已完成：

- 删除旧的带用户名人脸登录 Controller 接口；
- 删除旧的 `AuthService.faceLogin(username, file)`；
- 删除旧的模型客户端 `compareFace` 调用；
- 删除 `/api/auth/face/login` 的 Security 放行配置；
- 保留并启用 `/api/auth/face/login-auto`；
- 自动登录统一通过 `/face/distance` 计算最小距离匹配用户。
